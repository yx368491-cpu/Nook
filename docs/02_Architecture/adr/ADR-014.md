# ADR-014 · 本地缓存 Dexie + Outbox + client_msg_id Dedupe

## Status

✅ **Accepted**

## Date

2026-06-27

## Context

Nook 需支持弱网 / 离线场景。用户在网络断开时仍应能浏览历史消息、编辑 composer，网络恢复后自动同步。此外，同一消息可能因网络重试而被发送多次。

## Decision

**3 层缓存架构**：
1. **Hot（内存）** → Zustand · 最近 100 条 messages + 1 个 active conv draft。
2. **Warm（IndexedDB via Dexie）** → 最近 1000 条 messages + outbox + profile cache。
3. **Cold（服务端）** → Postgres SSOT。

**Outbox 流程**：
- 网络断：消息写入 Dexie outbox（`client_msg_id` + `state: 'pending'`）。
- 网络回：Service Worker background sync → 顺序 replay → 服务端 `client_msg_id` 去重。

**Dedupe**：
- 服务端：`messages_client_msg_id_unique_idx (conversation_id, client_msg_id) WHERE client_msg_id IS NOT NULL`。
- 客户端：相同 `client_msg_id` 的 Realtime INSERT + 本地 optimistic → 按 server-issued id 替换。

## Alternatives Considered

| 方案 | 优点 | 缺点 |
|------|------|------|
| **Dexie + outbox（选中）** | schema 迁移友好；比 raw IDB 简单 10× | 额外 10KB 依赖 |
| **raw IndexedDB** | 零依赖 | API 复杂；无版本管理 |
| **localStorage** | 简单 | 5MB 限制；同步 API 阻塞主线程 |
| **仅服务端** | 零客户端代码 | 无离线体验 |

## Consequences

- M5 实施；网络完全断开时 app 仍可浏览历史 + Composer 可输入。
- Dexie 自动版本管理（`db.version(1).stores(...)`）。
- `client_msg_id` 在 conv scope 内唯一，不同 conv 不冲突。

## 是否允许未来修改

**低可逆性**——移除 Dexie 需重新实现 IndexedDB schema + outbox + dedupe。
