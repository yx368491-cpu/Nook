# ADR-009 · 前端选型 React 18 + Vite + TypeScript

## Status

✅ **Accepted**

## Date

2026-06-27

## Context

Nook 是一个纯聊天 SPA（Single Page Application），不需要 SSR。前端选型需要考虑：启动速度、HMR 体验、类型安全性、生态成熟度、长期维护成本。

## Decision

**React 18 + Vite 5 + TypeScript**（纯 SPA，不选 Next.js / Astro / Remix）。

- 路由：React Router v6。
- 样式：Tailwind v3 + Nook-DESIGN-TOKENS.ts 注入。
- 构建：`npm run build` → `dist/` → CF Pages deploy。

## Alternatives Considered

| 方案 | 优点 | 缺点 |
|------|------|------|
| **Next.js** | SSR；SEO；流行 | 不需要 SSR；build 复杂；增加部署成本 |
| **React 18 + Vite（选中）** | 最轻量 SPA；HMR 最佳；CDN 部署无缝 | 无 SSR（Nook 不需要） |
| **Astro** | 岛屿架构；零 JS 默认 | chat SPA 核心场景需要大量 client JS |
| **SvelteKit** | Svelte 性能好 | 团队不熟悉；生态不如 React |

## Consequences

- 所有前端代码按 SPA 写（无 SSR 逻辑）。
- 单 repo 不上 monorepo（v1.0 简化）。
- CF Pages 部署 = `wrangler pages deploy dist/`。

## 是否允许未来修改

**中可逆性**——迁移成本大（如从 SPA 改 Next.js 需重写数据获取模式），但不需要「立即」做。
