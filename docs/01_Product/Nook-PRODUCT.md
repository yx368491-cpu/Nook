# Nook · 产品规划 v1.0

> 在 `Nook-DESIGN.md` 已经锁定的 UI 风格之上，给出产品的全部定义。
> 用途：作为后续功能开发、优先级排序和范围控制的基线。
> 风格：极度主观、观点鲜明、绝不含糊。

---

## 0. 一句话定位（放在所有内容之前）

**Nook = 你和密友的"深夜书房"，屏蔽掉整个互联网社交噪音的一间小房间。**

它不是商业产品，不是竞品，不是另一个微信。
它**不连接世界，它隔绝世界**。

---

## 1. 核心产品张力 · 及其解法

在做任何决定前，先承认并解决 Nook 的 4 个内生张力：

### 张力一：既然是"私人 + 密友"，为什么还要做跨端（PC + 移动）？
- **解法**：跨端不是为了 24 小时在线，而是为了**让对话像水一样无缝延续**——工作时 PC 浮岛里半摸鱼回复，睡前进被窝手机继续。
- **代价接受**：需要 invite-only + 加密传输 + 极简同步。
- **取舍**：30 天消息自动清理（不沉淀无限历史），降低数据负担，强化"此刻 living in the moment"的产品调性。

### 张力二：极简 vs. 真实聊天需要很多功能
- **解法**：保留交流的**骨架**，砍掉效率的**肌肉**。
  - ✅ 文本 + 图片 + 极少量 emoji 反应 = 够了
  - ❌ 群公告、接龙、机器人、富文本面板 = 不要
- **判断准则**：每一个候选功能问自己——"加了之后，我们这十几个人会更愿意在深夜打开它吗？"
  - 如果答案是"更愿意"，留。
  - 如果答案是"功能上更全"，砍。

### 张力三：科技感 + 年轻化 vs. 受众只有十几个人
- **解法**：科技感通过**隐形参数**体现，**绝对不外露**。
  - ss03 字体细节、180 ms 入场缓动、WebSockets 极低延迟 → 这些是用户"感觉得到但说不出"的东西。
  - 不暴露：斜杠命令、Markdown 快捷键、API 文档链接、技术博客角标。

### 张力四：通讯是个红海市场，Nook 凭什么存在？
- **解法**：Nook 不与微信、Discord、Telegram 比功能完整度。
- 它与所有主流 IM 比的是**信号噪声比**——在这里，每一条弹出的消息都 100% 是你真正想看的。

> 一句话总结这四个张力：Nook 是一种**有意识地选择回避**的产品，与所有想"做得更多"的 IM 反向而行。

## 2. 反模式清单（**做 Nook 必须坚决避免**的）

既然这是私人 + 朋友的产品，**主流 IM 都做的事，也许恰恰是 Nook 不能做的事**：

| 反模式 | 来自 | 为什么必须避免 |
|---|---|---|
| **已读回执（蓝色对勾/I、II）** | 微信 / iMessage | 让小圈子也有"被凝视感"，破坏"温柔的默契"。看见即回复是朋友之间的事，不需要软件强制。 |
| **红色未读小角标** | 微信 / 钉钉 | 红色的攻击性太强。Nook 用未读会话列表项的微弱 accent-soft-bg 替代。 |
| **语音消息（按住说话、小绿条）** | 微信 | 私人场合下对接收者的不尊重行为，且破坏深色阅读节奏。 |
| **表情包商店 / Sticker Market** | 微信 / iMessage | 让一个私人空间变成商场。 |
| **多人频道 / 服务器分区** | Discord / Slack | 朋友之间不需要"工作空间"的层级感。 |
| **视频/语音通话按钮** | 微信 / Telegram | 通话会立刻把所有美学拉回 Zoom 质感。想打电话的人切回电话或微信。 |
| **"发现"/"小程序"/"工作流"** | 微信 / 企业 IM | 主界面只有一栏：对话。没有别的。 |
| **浅色模式切换** | 所有 IM | 强制深色不仅是省事，更是**建立打开 Nook = 进入黑夜**的空间心智。 |
| **群公告 / 群待办** | 微信 / 钉钉 | 朋友群 ≠ 工作群，不需要治理工具。 |
| **朋友圈 / 动态墙** | 微信 | 禁止扩展成"社交平台"。 |
| **加好友 / 二维码名片** | 微信 | Invite-only 邀请制永远是唯一入口。 |
| **已登录设备管理（多端同步显示在线）** | 微信 | 我们都是朋友，谁登了几台不需要汇报。 |

> 这张表本身就是 Nook 品牌的一部分。可以打印出来，贴在显示器旁边。

## 3. 产品定位 / 目标 / 核心价值 / 用户画像 / 使用场景 / 边界

### 3.1 产品定位（Positioning）

> **"少数密友的数字避难所"**（A Digital Sanctuary）

- 没有商业目的：从不考虑规模化、不考虑商业化、不考虑上市。
- 平衡PC端和移动端的使用体验

### 3.2 产品目标（Goals）

按优先级排：

1. **即刻沉浸**（打开 → 进入房间 → 阅读 → 关闭，全程没有任何摩擦）。
2. **完全隔离**（与微信/工作场景在视觉和心智上泾渭分明——打开 Nook 是"脱下社会身份外套"）。
3. **视觉愉悦**（每一次击键、每一条气泡出现的动效都是感官享受）。
4. **节奏感**（强调整体呼吸、留白、节拍，而不是功能密度）。
5. **陪伴感**（不是"消息工具"，是"我们几个此刻都在这条流里"的感觉）。

### 3.3 核心价值（Core Value Proposition）

> **零噪音的信息获取权。**

弹出的每一条消息都**绝对**是你关心的。
**没有**订阅号、群发助手、推送营销、机器人提醒、广告。
**没有**"已读"的压力。
**没有**用来"看起来很忙"的状态表演。
**100%** 的信噪比。

### 3.4 用户画像（User Persona）

#### 一级画像 · 你（产品发起人）

- 20 岁，对 UI/设计有较高敏感度。
- "微信疲劳症"中重度患者——更喜欢简短的、节奏合得来的、与朋友圈完全分割的私人表达。
- 技术背景或对技术好奇，所以才愿意让 AI agent 帮他写一份"产品设计规范"。
- 文字表达力强（很可能是写代码 / 设计 / 文案 / 学生）。

#### 二级画像 · 你的少数朋友（5–15 人）

- 与你年龄相仿、生活节奏相似。
- 接受纯文字 + 偶尔图片的交流节奏（不会因为没有表情包而退场）。
- 不需要 Nook 来取代工作场景，他们自己的微信工作已经够多了。
- 对"我们几个有一个专属空间"这件事**心里存着隐性渴望**——但不会主动说出来。

#### 排除画像（**反画像**· 不该成为 Nook 用户的）

- ❌ 一个无法忍受纯文字的人
- ❌ 期待语音/视频通话的人
- ❌ 习惯微信群"@所有人"的强势表达者
- ❌ 把 Nook 当作"备份微信"的人

### 3.5 使用场景（Use Cases）· 真实检验

> 现实问题：你们这几个人在微信上也有个群，凭什么会跑来 Nook？

3 个真实且高频的场景，给 Nook 的"存在的理由"：

#### 场景 A：深夜无感灌水（Late-night Banter）
- 凌晨 1 点，你睡不着，想起一个朋友今天遇到的好笑事。
- 微信里有 47 个未读工作消息，目标好友在第 13 个对话里——你不想翻。
- 打开 Nook，只有 4 个朋友在线，他们都在流里，**直接发，不用犹豫**。
- 不需要表情包的喧嚣——只需要干净的字 + 一两张原图。

#### 场景 B：高清原图流（High-fidelity Sharing）
- 上周末聚会，朋友拍了 200 张照片，要在群里分享。
- 微信原图压缩、9 张限制、动辄变成"压缩包接力赛"。
- 在 Nook，发原图，一张一张看到原色原质感，配上简短的文字点评。
- **Nook 就是你们的高级影集流**。

#### 场景 C：赛博陪伴（Ambient Co-presence）· 这是 Nook 的灵魂场景
- 你们这群人可能好几天都不说话。
- 但打开 Nook，侧栏里朋友名字旁**缓慢呼吸的 lavender blue 小点**，偶尔谁打几个字又撤回。
- Typing 三点持续若干分钟，最后落下一条"晚安"。
- **你们不需要聊天，你们需要的只是"彼此都在这条流里"的事实本身。**

#### 边界场景（**不该是正常场景**的）

- ❌ 群体投票 / 在线文档协作（用专门的工具）
- ❌ 临时与陌生人建立联系（用专门的工具）
- ❌ 工作汇报、招投标、客户沟通（用微信/邮件）

### 3.6 产品边界（Scope）

#### 在边界内（v1.x 做）

- ✅ 一对一 / 多人**纯文字聊天**
- ✅ **单张**原图、原文件分享（高保真、无压缩）
- ✅ 极少量 **emoji 反应**（限 6 个最底层系统 emoji）
- ✅ **消息回复 / 撤回 / 删除 / 编辑（限时 2 分钟）**
- ✅ **Typing 打字指示器**（灵魂功能，必须做精）
- ✅ **Ambient 在线状态**（火炉的火光，限侧栏名字旁的小点）
- ✅ **全端同步**（PC + 移动）
- ✅ **30 天消息自动清理**（不沉淀无限历史）

#### 永远不在边界内（v∞ 也不做）

- ❌ 语音消息
- ❌ 语音通话 / 视频通话
- ❌ 表情包商店 / Sticker Market
- ❌ 朋友圈 / 动态墙 / 个人主页
- ❌ 群公告 / 群待办 / 群管理工具
- ❌ 二维码加好友 / 通讯录导入
- ❌ 多设备登录管理
- ❌ 钱 / 支付 / 红包

#### 待定（v1.1+ 看朋友反馈决定）

- 🟡 简单的全局搜索
- 🟡 转发消息
- 🟡 成员仅邀请制简单群组（不再支持创建大量群组）
- 🟡 链接预览（带可选关闭，避免被滥用）

## 4. 功能分级裁决（Must / Should / Future）

> 没有"看情况"。只有"做"或"不做"。
> 如果犹豫，就降一级（Should → Future，或直接砍）。

### 🔴 MUST HAVE · 没它 Nook 不能叫产品

| # | 功能 | 为什么必须 |
|---|---|---|
| M1 | **1:1 文字单聊** | 核心基石。 |
| M2 | **核心小群聊（最多 4 个群）** | 一群人共享同一个空间。数量克制是品牌的一部分。 |
| M3 | **Typing 打字指示器** | Nook 的灵魂功能——看到朋友在打字本身就有价值。做得**极优雅**（降速三点，120 ms 错落）。 |
| M4 | **消息回复 / 引用** | 群聊里不指代清楚会鸡同鸭讲。配合 composer 上方的 `surface-2` 引用卡。 |
| M5 | **消息撤回** | 给"深夜冲动发言"一个软后退的余地。 |
| M6 | **消息删除** | 与撤回不同——删除是"我不想让你再看到这条"；撤回是"我反悔了"。 |
| M7 | **原图分享（单张）** | 高保真是 v1 不可妥协的承诺。多次滚动里每次只发一张最敬畏。 |
| M8 | **原文件分享（≤ 50 MB）** | 一次传送一份大文件，不打散。 |
| M9 | **Invite-only 邀请制** | 入口是品牌。**不允许任何其他注册途径**。 |
| M10 | **全端同步（PC + 移动 + Web）** | 工作 PC → 道路移动 → 睡前手机，对话无缝。 |
| M11 | **消息持久化与跨端拉取** | 一句老话：30 天内可读，过期自动清理。 |

### 🟡 SHOULD HAVE · 拉开品质差距的"魔法"

| # | 功能 | 为什么 Should |
|---|---|---|
| S1 | **Ambient 在线状态** | 朋友名字旁缓慢呼吸的 6 px lavender blue 小点。这是"避难所"里火炉的火光。**绝不做打字或已读状态**——只做"在 / 不在"。 |
| S2 | **基础 Emoji 反应**（限 6 个：👍❤️😂👀🔥🙏） | "已查阅"的默契替代。**不开放完整键盘选择器**——只能从这 6 个里点。 |
| S3 | **消息编辑（限时 2 分钟）** | 高级感容不下错别字。带极小的 `(edited)` 印记。 |
| S4 | **断网重连与状态指示** | 网络抖动时右下角出现极小 signal light，颜色从 success → warning，而不是闪烁红 banner。 |
| S5 | **消息时间分组（今天 / 昨天 / 本周）** | 长对话起源节奏的本能需求。 |
| S6 | **空状态引导** | 第一次进入没有朋友时的极简 onboarding 提示，避免冷启动孤独感。 |
| S7 | **设置抽屉（含深色强度调节 / 通知开关 / 已退账户）** | 极少使用，但需要存在。 |

### 🟢 FUTURE · 现在不做，等朋友真的需要时再做

| # | 功能 | 为什么 Future |
|---|---|---|
| F1 | **全局搜索** | 数据量在头几周太低，无意义。后期再加。 |
| F2 | **消息转发** | 暂不需要，**且容易让对话变得不再亲密**。 |
| F3 | **消息引用链接（Permalink）** | 后期可做，但 v1 不做。 |
| F4 | **Pin / 星标消息** | 暂不需要。 |
| F5 | **未读消息标记**（用柔和淡化替代红点） | 需要先做完基础聊天再考虑。 |
| F6 | **链接预览** | 防止被滥用为 SEO / 群发工具，**默认关闭**，未来按人开放。 |
| F7 | **iOS / Android 原生 App（壳应用）** | PWA 已能满足大部分需求，原生仅在体验提升显著时考虑。 |
| F8 | **端到端加密（E2EE）** | v1.1 计划，朋友足够信任时再升级。 |
| F9 | **本地消息缓存 / 离线阅读** | 渐进式优化，v1 简单云优先即可。 |
| F10 | **多群支持 / 大群（>10 人）** | 严格不鼓励。两个核心群 + 几个 1:1 已经够。 |

| N2 | ~~语音消息~~ | 视觉美感破坏器。 |
| N3 | ~~语音 / 视频通话~~ | 拉低了产品美学一致性，且超出 MVV 能力。 |
| N4 | ~~表情包商店 / Sticker Market~~ | 与"避难所"调性冲突。 |
| N5 | ~~朋友圈 / 动态墙~~ | 不要扩展为社交平台。 |
| N6 | ~~群公告 / 群管理~~ | 朋友群 ≠ 工作群。 |
| N7 | ~~多设备登录管理 / 设备列表~~ | 大家都是朋友，不需要透明化彼此的设备。 |

## 5. v1 路线图（执行节奏建议）

> 这里给的只是"顺序"，不是"完整 PRD"。具体 PRD 在每个版本独立另写。

### v1.0 · MVP（约 4–6 周个人工作量）

**目标**：能让 5 个朋友至少进得来、发得出、看得见。

按 Must Have 列表实现 M1–M12，去掉一切花哨。
- 单色阶仅确定 (`#0F1115` canvas + `#7B85F0` accent)
- 完成一个 web 端 + 一个 PWA 移动端
- Invite-only 用一次性邀请链接
- 消息持久化用 SQLite / LiteFS（小型个人项目够用）
- 部署到一台小型 VPS（1 GB RAM 即可）

### v1.1 · 灵魂打磨（上 MVP 后 2–3 周）

- 大幅琢磨 Typing 动画
- 优化 composer 浮岛的输入体验
- 加入 Ambient 在线状态（呼吸光点，S1）
- 加入 6 个 emoji 反应（S2）

### v1.2 · 容器升级（让朋友再多几个进来）

- 加入消息编辑 + `(edited)` 标记（S3）
- 加入消息分组（今天 / 昨天 / 本周，S5）
- 加入断网 status light（S4）

### v2.0 · 可选架构升级

- 端到端加密（如果朋友提了"我担心消息安全"）
- 原生 App 壳（如果 PWA 体验有显著问题）
- 多设备同时在线的冲突解决

## 6. 成功度量（这个产品的"OKR"）

> 一个私人 + 小圈子产品，**不应该**用 DAU / MAU / retention 衡量。
> 它应该用 _人心_ 衡量。

| 心智指标 | "成功"的标志 |
|---|---|
| **是否打开** | 朋友说"我今天睡前想看一眼 Nook"——产品融入了睡前仪式。 |
| **是否敢说** | 朋友说"我那天发的朋友圈太装了，但我在 Nook 里发的是真实的"——产品帮你卸下"社会身份外套"。 |
| **是否陪伴** | 朋友说"我知道你们都在那条流里"——即使你们几天没说话也不觉得断了。 |
| **是否悦目** | 你自己每周打开 Nook 的时候，仍然觉得"哇，好看"——产品没有让你审美疲劳。 |
| **是否省心** | 你 N 个月内**不需要**为它写代码、做 bugfix、维护、救火——产品已经稳定。 |

> 如果 Nook 需要不断推新功能来维持，那它就失败了。
> Nook 成功的标志，是你在 6 个月后不想改它。

## 7. 决策清单（开发期随时对照）

任何一次新功能讨论前，**问自己这 4 个问题**：

1. **这个功能让我朋友"更愿意在深夜打开 Nook"吗？** 没回答 YES → 砍。
2. **这个功能有没有给"已读回执"让路？** 有 → 砍。
3. **这个功能是不是某个主流 IM 也有的？** 是 → 警惕，可能不该做。
4. **这个功能如果做得不够完美，会破坏现有美学吗？** 会 → 推迟或砍。

> 这 4 个问题应该写在 README 第一页。
