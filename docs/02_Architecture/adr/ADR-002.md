# ADR-002 · i18n v1.0 仅双语（zh-CN + en）

## Status

✅ **Accepted**

## Date

2026-06-27

## 修正历史

- v1.0.1 docs-only patch（FU-2 sync）— 从 3 语裁决为 2 语

## Context

`../Nook-ARCHITECTURE.md` § 2.7 原始设计为 3 语支持（zh-CN / en / ja-JP）。`../../01_Product/Nook-INTERVIEW-spec.md` Round-3 Q2 以及 `../../01_Product/Nook-SPEC.md` § 1.8 裁决为双语。产品定位 20 岁左右、私人朋友圈层，用户场景不需要 ja-JP。

## Decision

**v1.0 上线仅双语**：zh-CN + en。ja-JP 等 v1.1+ 视用户需求再加。

影响范围：
- 架构层仍按 dynamic locales design（i18next），ja-JP 加语言 = +1 个 JSON 文件。
- M1-3 i18next locales 只有 2 个 JSON。
- REACT-I18N fallback zh-CN。
- UI 文案 100% 走 key 而不是硬编码。

## Alternatives Considered

| 方案 | 优点 | 缺点 |
|------|------|------|
| **直接 3 语（zh-CN/en/ja-JP）** | 未来不需要再改 | ja-JP 的翻译流水线、RTL 兼容性 ≠ 0 维护成本；v1.0 不为「未来」付维护成本 |
| **仅 zh-CN 单语** | 维护成本最低 | 无法满足产品定位中英混用的核心场景 |

## Consequences

- 翻译维护量最小化。
- 架构仍保留多语言扩展能力（i18next）。
- 加 ja-JP 只需新增文件，无需改代码。

## 是否允许未来修改

**低可逆性**——加语言可以（+1 文件），但减语言不可逆。v1.1+ 如有 ja-JP 需求，走新版本。
