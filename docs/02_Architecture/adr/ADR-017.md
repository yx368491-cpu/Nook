# ADR-017 · 监控 Sentry free + LogSnag free

## Status

✅ **Accepted**

## Date

2026-06-27

## Context

Nook 需要前端错误监控、性能追踪、结构化事件日志。作为永久免费项目，监控工具需在免费层内满足需求。

## Decision

- **Sentry**（free tier：5k errors/月）→ 异常 + performance 监控。
- **LogSnag**（free tier：1k events/月）→ 结构化事件 log（登录、邀请、清理 cron 等）。

**隐私红线**：
- Sentry 关 `sendDefaultPii: false`。
- message body 和 email **永不进入**任何监控/日志系统。
- LogSnag 事件仅含 `{ event, conversation_id_hash, kind, ts }` 结构。

## Alternatives Considered

| 方案 | 优点 | 缺点 |
|------|------|------|
| **Sentry + LogSnag（选中）** | 各自职责清晰；免费层匹配用量 | 两个 dashboard |
| **Datadog** | 全栈可观测性 | 超出预算 |
| **仅 Supabase 内置日志** | 零配置 | 仅 7 天保留；缺少前端监控 |
| **Sentry + 自写 app_events 表** | 完全控制 | 维护成本高 |

## Consequences

- Sentry 免费 5k error/月足够 Nook 1 年用量。
- LogSnag 1k event/月约 300/mo（登陆 + 邀请 + cron 等）。
- 7 天内 Supabase 日志 + 90 天 app_events 表 + 180 天冷归档 R2。

## 是否允许未来修改

**高可逆性**——可迁 Datadog / OpenTelemetry 自管，但免费层是本阶段上限。
