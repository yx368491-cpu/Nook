# ADR-016 · CI/CD = GitHub Actions

## Status

✅ **Accepted**

## Date

2026-06-27

## Context

Nook 需要 CI/CD 流水线：PR 验证（typecheck/lint/test）+ main 自动部署（FE + EF）。需要低成本、与 GitHub 集成的方案。

## Decision

**GitHub Actions**（2000 min/月免费，Nook 约 50 min/月）。

流水线：
1. PR 验证：`typecheck` → `lint` → `test` → `supabase db push --dry-run`。
2. main 分支：`verify` → `wrangler pages deploy dist/` → `supabase functions deploy`。
3. 部署审批：DB migration 走手动审批（避免意外 DDL）。

## Alternatives Considered

| 方案 | 优点 | 缺点 |
|------|------|------|
| **GitHub Actions（选中）** | 2000 min/mo 免费；与 GitHub 内置 | — |
| **Cloudflare Pages CI** | 与 FE 部署集成 | 仅 FE；不能运行 EF deploy |
| **GitLab CI** | 流水线强大 | 需额外托管 |

## Consequences

- 每个 PR 必须通过 typecheck/lint/test。
- CD 阶段自动部署 FE + EF；DB migration 手动审批。
- 需要配置 `CF_API_TOKEN` / `SB_ACCESS_TOKEN` 等 secrets。

## 是否允许未来修改

**高可逆性**——可迁 GitLab CI / Jenkins / CircleCI，只需重写 workflow YAML。
