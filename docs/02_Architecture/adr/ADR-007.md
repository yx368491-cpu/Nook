# ADR-007 · 编辑 2 分钟窗口 = DB trigger

## Status

✅ **Accepted**

## Date

2026-06-27

## Context

`../../01_Product/Nook-SPEC.md` § 2.3 F-MSG-05 规定消息编辑必须在 2 分钟内完成。该约束需要是 DB-authoritative：即使用户通过 API 直接修改，后端也必须拒绝超时编辑。

## Decision

`trg_messages_edit_window` BEFORE UPDATE on messages 检测 `OLD.created_at < now() - interval '2 minutes'` 且 `NEW.body is distinct from OLD.body` 时 `raise exception 'EDIT_WINDOW_EXPIRED'`。

- trigger 自动设置 `NEW.edited_at := now()`（客户端不可伪造）。
- UI 层：编辑按钮在 2 分钟后 disabled + tooltip 提示。
- 列级 GRANT：仅允许 update `(body, deleted_by_sender_at)`；其他列不可更新。

## Alternatives Considered

| 方案 | 优点 | 缺点 |
|------|------|------|
| **仅前端 disabled** | 实现快 | 用户可通过 API 绕过 |
| **DB trigger + 应用层 check（选中）** | 双层防线 | trigger 额外开销极低 |

## Consequences

- 任何超过 2 分钟的编辑请求都会被 DB 拒绝。
- 客户端必须在 mutation 前检查时间窗（优化体验），但不能依赖此检查。
- `edited_at` 字段由 trigger 自动设置，客户端无需传入。

## 是否允许未来修改

**高可逆性**——任意时刻可修改时间窗口长度或移除 trigger。
