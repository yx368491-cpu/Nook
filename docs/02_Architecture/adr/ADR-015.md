# ADR-015 · 部署 Cloudflare Pages + Supabase Cloud + R2 fallback

## Status

✅ **Accepted**

## Date

2026-06-27

## Context

Nook 遵循 P-01：「永久免费 > 过度设计」。需要选择满足免费 + 可靠 + 低维护成本的部署方案。Vercel（严格 Hobby 限）和 Netlify（冷启动）不适合 Nook。

## Decision

**3 层永久免费部署**：

| 层 | 服务 | 免费限 | Nook 用量 |
|---|---|---|---|
| FE | Cloudflare Pages | 无带宽上限 | < 5 GB/月 |
| BE | Supabase Cloud | 500MB DB / 1GB Storage / 2GB 带宽/月 / 200 WS 连接 | 远低于限 |
| 兜底 | Cloudflare R2 | 10GB 存储 / 1M ops/月 | v1.0 不启用 |

## Alternatives Considered

| 方案 | 优点 | 缺点 |
|------|------|------|
| **Vercel + Supabase** | 流行；部署快 | Vercel Hobby 限（100GB 带宽 / 60 builds/h） |
| **CF Pages + Supabase（选中）** | 无带宽上限；单 dashboard | 无 SSR（Nook 不需要） |
| **Railway + MinIO** | 灵活 | 免费层有限；复杂度高 |
| **自管 VPS** | 完全控制 | 运维成本高 |

## Consequences

- 避开 Vercel 严格 Hobby 限 + Netlify 冷启动。
- CF Pages 自动 HTTPS + 全球 PoP。
- Supabase Storage 是唯一 6-8 月可能逼近的资源（KI-1，v1.1 准备迁 R2）。

## 是否允许未来修改

**中可逆性**——可迁 Vercel / Netlify / 自管，但需要重配部署流水线。
