# ADR-010 · 后端 Supabase 一体化

## Status

✅ **Accepted**

## Date

2026-06-27

## Context

Nook 是小规模私人聊天系统（≤ 20 好友），后端需要提供：DB + Auth + Realtime + Storage + Serverless Functions。按 P-01「永久免费 > 过度设计」，不应引入多供应商拼装方案。

## Decision

**Supabase 一体化**：一家供应商承担后端全部 5 层。

- Postgres 15 + pg_cron。
- Auth (GoTrue) + JWT。
- Realtime (WS + Presence)。
- Storage + signed URL。
- Edge Functions (Deno)。

## Alternatives Considered

| 方案 | 优点 | 缺点 |
|------|------|------|
| **Supabase 一体（选中）** | 单 dashboard、单 RBAC、单 billing；维护成本 ≈ 0 | 单厂商故障 = 全栈停 |
| **Pusher + Pocketbase + AWS S3** | 各层独立 | 多 dashboard、多故障点、超出免费层 |
| **Firebase** | 成熟；免费层大 | 供应商锁定更强；中国大陆不可达 |
| **自管 VPS（Postgres + WS + MinIO）** | 完全控制 | 维护成本高；运维负担重 |

## Consequences

- 单厂商故障 = 全栈停（KI-4，`../../03_Engineering/KNOWN_ISSUES.md`）。
- 不会引入 Pusher / Ably / Firebase / Pocketbase。
- v2.0+ 可评估自托管逃生方案。

## 是否允许未来修改

**低可逆性**——迁移成本大（需重新实现 auth/realtime/storage 三层）。v2.0+ 自管备选。
